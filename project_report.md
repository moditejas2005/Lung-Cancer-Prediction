# Synthetic Medical AI Project Report

This document provides a detailed overview of the project structure, including what each file and directory contains. Unwanted files (like `__pycache__` directories) have been successfully removed to keep the project clean.

## Root Directory (`synthetic_medical_ai/`)

- **`app.py`**: The main entry point for the web application (likely using Flask or FastAPI), handling API routes and serving the frontend.
- **`pipeline_runner.py`**: A script to execute the end-to-end machine learning pipeline, combining data generation, preprocessing, training, and evaluation.
- **`inference.py`**: Contains functions or classes responsible for making predictions using the trained machine learning models.
- **`generate_excel_reports.py`**: A utility script designed to generate detailed Excel reports based on the model's predictions, validations, or synthetic data generation metrics.
- **`validators.py`**: Contains data validation logic to ensure inputs to the model or pipeline meet the required formats and constraints.
- **`requirements.txt`**: Lists all Python dependencies required to run the project.
- **`README.md`**: The main documentation file explaining the project's purpose, setup instructions, and usage.

## 1. `augmentation/`
Handles the generation of synthetic medical data using generative models (like GANs).
- **`conditional_sampling.py`**: Logic for sampling synthetic data based on specific conditions or classes.
- **`ctgan_trainer.py`**: Script for training a CTGAN (Conditional Tabular GAN) model on tabular medical data.
- **`gan_monitor.py`**: Tools for monitoring the training process of the GAN models.
- **`synthetic_generator.py`**: The main interface for generating synthetic data once the GAN models are trained.

## 2. `config/`
Contains configuration files for the project.
- **`settings.json`**: General configuration parameters, thresholds, and settings used across the application.

## 3. `data/`
The directory where datasets are stored. It contains the following subdirectories:
- **`raw/`**: Original, unmodified data.
- **`processed/`**: Cleaned and preprocessed data ready for training.
- **`synthetic/`**: Generated synthetic data from the augmentation pipeline.
- **`validated/`**: Data that has passed quality and validation checks.
- **`reports/`**: Stored reports generated by the pipeline.

## 4. `explainability/`
Contains scripts dedicated to model explainability and interpretability, crucial for medical AI.
- **`feature_importance.py`**: Calculates and visualizes the global importance of features used by the models.
- **`prediction_explainer.py`**: Provides explanations for individual predictions.
- **`shap_analysis.py`**: Uses SHAP (SHapley Additive exPlanations) values to explain the output of the machine learning models.

## 5. `logs/`
Stores log files generated during application runtime and pipeline execution.
- **`pipeline.log`**: Detailed logs from the pipeline runner.

## 6. `models/`
Stores serialized (saved) machine learning models and feature schemas.
- **`best_xgb_model.joblib`**: The best performing XGBoost model.
- **`calibrated_stacking_model.joblib`**: A calibrated stacking ensemble model.
- **`ctgan_model.joblib`**: The trained CTGAN model for data generation.
- **`stacking_ensemble.joblib`**: The base stacking ensemble model before calibration.
- **`preprocessing_pipeline.joblib`**: The fitted data preprocessing pipeline (scalers, encoders, etc.).
- **`calibrated_stacking_features.json` & `calibrated_stacking_original_features.json`**: JSON schemas detailing the features used by the models.
- **`threshold_config.json`**: Optimized threshold settings for classification.
- **`old_model/`**: Archive of previously trained models.

## 7. `preprocessing/`
Scripts dedicated to preparing raw data for machine learning.
- **`data_cleaner.py`**: Handles missing values, outliers, and data formatting.
- **`feature_engineering.py`**: Creates new features from existing data to improve model performance.
- **`preprocessing_pipeline.py`**: Orchestrates the cleaning and engineering steps into a reusable pipeline.

## 8. `templates/`
Contains the HTML templates for the web interface.
- **`index.html`**: The main landing page or input form for the medical application.
- **`result.html`**: The page displaying the prediction results and explainability metrics.

## 9. `training/`
Scripts responsible for training the machine learning models.
- **`train_models.py`**: The main script to initiate the training of base models.
- **`ensemble_pipeline.py`**: Logic for combining multiple models into a stacking ensemble.
- **`calibration_pipeline.py`**: Calibrates the output probabilities of the trained models to ensure they represent true likelihoods.
- **`hyperparameter_tuning.py`**: Script for searching the optimal hyperparameters for the models.
- **`threshold_optimizer.py`**: Optimizes classification thresholds to maximize specific medical metrics (e.g., Recall/Sensitivity).

## 10. `validation/`
Ensures the quality of the data and the generated synthetic datasets.
- **`medical_validator.py`**: Validates data against known medical rules and constraints to ensure clinical realism.
- **`statistical_validator.py`**: Performs statistical tests to compare distributions between real and synthetic data.
- **`drift_detector.py`**: Detects data drift over time, which might degrade model performance.
- **`quality_scoring.py`**: Computes an overall quality score for the synthetic datasets.
